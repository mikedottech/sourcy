Analyze tree grammar:

Types of arguments:
	- Int
	- Float
	- ID
	- Number (Int or Float)

1.- TRIGGERCALLS (Regular syntax)
	- if the name of the trigger exists then
		- if the number of arguments match then
			- if the type of arguments match then
				- ok
			- else
				- if expected int and found float or the other way around then
					- Log warning
					- Coerce rewriting tree
				- else
					- Log error
					- Abort
				- end if
			- end if
		- else
			- Log error: The trigger X does not take Y arguments
		- end if
	- else
		- if the context is right (rhs expecting an ID) then
			- Rewrite tree to remove triggercall
		- else
			- Log error: Trigger X not found
		- end if
	- end if
	
-(Old-style, whitespace and irregular syntax require special caring)

2.- STATES
	- Make sure they specify all the mandatory attributes
	- Make sure the attributes RHS match

3.- STATE CONTROLLERS
	- For each state type
		- Make sure they specify the required arguments
		- Make sure the attributes RHS match
		
-=BETWEEN FILES=-
	- Log a warning if a state is overriden

().- Check that states we are jumping to exist

function statedef_200()

-- State assignments

-- State 0, REMOVEEXPLOD
if time() == 0 then
	removeExplod(
		{ 
			id = 200 
		}
	)
end

-- State 1, POSADD
if animElem() == 2 then
	posAdd(
		{
			x = -15
		}
	)
end if

-- [...]

-- State 0, EXPLOD
if animElem() == 6 then
	explod(
		{
			anim = 201,
			id = 200,
			pos = point{0,0},
			posType = "P1",
			sprPriority = 3,
			removeOnGetHit = 1,
			ignoreHitPause = 0,
			bindTime = -1
		}
	)
	
end

-- State 0, PLAYSND
if animElem() == 5 then
	playSnd(
		{
			value = {"S", 13, 12}
		}
	)
end

end
